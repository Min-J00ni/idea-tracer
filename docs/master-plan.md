## 1. Executive Summary

IdeaTracer는 회의 녹음을 **실시간 인터랙티브 마인드맵**으로 변환하는 AI 기반 회의 인사이트 플랫폼이다.

기존 도구들(Otter.ai, Fireflies, tl;dv, 클로바노트)은 음성을 텍스트로 변환하는 데 그친다. 사용자는 여전히 수십 페이지의 전사본을 읽어야 하며, 팀에 공유해도 아무도 읽지 않는다. 마인드맵 도구(Mapify, Xmind, Ayoa)는 시각화가 강점이지만 실시간 회의 봇이 없고 타임스탬프 연결이 불완전하다.

**IdeaTracer는 두 영역의 공백을 동시에 채운다**: 실시간 회의 봇 + 타임스탬프 연결 마인드맵의 조합. 현존하는 어떤 도구도 이 두 가지를 동시에 완성하지 못하고 있다.

**목표 시장**: 회의·인터뷰를 분석·공유해야 하는 지식 노동자 — UX 리서처, 스타트업 PM, 컨설턴트를 초기 타겟으로 삼는다.

## 2. 문제 정의 — 왜 지금인가

### 2.1 정량화된 고통

| 지표 | 수치 | 출처 |
|------|------|------|
| 직장인 주간 회의 시간 | 7.5시간 | Microsoft Work Trend Index 2023 |
| 정보 과부하 경제적 손실 | 연 $1조 (글로벌) | Rensselaer 공대, 2024 |
| AI 분석·합성 도구 사용 UX 리서처 | 90% | User Interviews AI Report 2024 |
| AI의 질적 분석 시간 단축 | 최대 80% | Loop11 UX Research Trends 2025 |
| 정보 과부하 일상 스트레스 호소 직장인 | 76% | 글로벌 설문 종합 |

### 2.2 현재 사용자 여정의 구조적 문제

```
회의 진행 → 봇 자동 녹음 → 텍스트 덩어리 생성
         → [고통점] 긴 스크롤, 구조 파악 불가
         → 팀 공유해도 아무도 안 읽음
         → 액션아이템 누락, 의사결정 재논의 반복
```

### 2.3 기존 도구들이 해결하지 못한 것

- **회의 전사 도구** (Otter, Fireflies, tl;dv, Notta): 텍스트 요약만 제공. 시각화 없음
- **마인드맵 도구** (Mapify, Xmind, Ayoa): 시각화 강점이나 실시간 봇 없음, 타임스탬프 연결 불완전
- **국내 도구** (클로바노트): 한국어 STT 최강이나 시각화 기능 전무
- **Notta**: 마인드맵을 제공하는 유일한 회의 도구이지만 자동 생성 구조에 종속, 커스텀 자유도 낮음

> **결론**: 봇 + 타임스탬프 마인드맵의 조합은 현존하는 어떤 도구도 완성하지 못한 공백이다.

## 3. 비전 & 미션

### 비전

> 모든 대화에서 가장 중요한 아이디어가 자동으로 추적되고, 누구나 이해할 수 있는 시각적 구조로 팀에 전달되는 세상

### 미션

> 회의와 인터뷰의 내용을 텍스트 덩어리가 아닌 **탐색 가능한 지식 지도**로 바꾸어, 개인과 팀이 더 빠르게 인사이트에 도달하게 한다

### 포지셔닝 문장

> "IdeaTracer는 AI 회의 전사 도구가 아니다.  
> **회의 인사이트의 시각적 구조화 플랫폼**이다."

*— "AI 마인드맵 도구"로 포지셔닝하면 Xmind·Mapify와 형식 비교 경쟁에 끌려 들어간다. "구조화된 인사이트"로 포지셔닝하면 전사 도구와도, 마인드맵 도구와도 직접 경쟁하지 않는다.*

## 4. 타겟 시장 & 페르소나

### 4.1 시장 규모 추정

- **AI 회의 어시스턴트 글로벌 시장**: 2024년 약 $1.9B → 2029년 $6.7B 예상 (CAGR ~28%)
- **1차 타겟 지역**: 한국 (클로바노트 공백 활용) + 영어권 확장
- **초기 집중 세그먼트**: UX 리서처, PM, 컨설턴트 — 이미 AI 분석 도구에 지출 중, 추가 지불 저항 낮음

### 4.2 페르소나 1 — UX 리서처 "서하은"

**프로필**: 32세, 스타트업 UX 리서처, 3인 소규모 팀

**현재 워크플로우**
- 주 4~6건 사용자 인터뷰 (각 40~60분)
- Otter.ai 전사 → 직접 Miro에서 어피니티 다이어그램 수동 작성
- 인터뷰 1건 분석에 평균 **2~3시간 소요**

**핵심 불만**
- "전사본은 무슨 말이 있었는지만 알려줘. 왜가 중요한지는 내가 찾아야 해"
- 스테이크홀더에게 긴 전사본 첨부는 비효율적이나 대안이 없다
- 10개 인터뷰에서 공통 패턴 추출하는 데 하루가 사라진다

**IdeaTracer 사용 시나리오**
1. 인터뷰 완료 → 자동 마인드맵으로 테마 구조 즉시 파악
2. 노드 클릭 → 발언 원문 + 타임스탬프로 즉시 이동
3. PM에게 마인드맵 링크 하나로 공유 (영상/텍스트 첨부 없음)
4. 5개 인터뷰 마인드맵을 한 화면에서 패턴 비교

**구매 패턴**: 개인 구독 → 팀 확장. 제품 적합도 **매우 높음**.

### 4.3 페르소나 2 — Product Manager "김준서"

**프로필**: 29세, Series A 스타트업 PM, 개발팀 8명 협업

**현재 워크플로우**
- 일 평균 5~7건 회의 (스탠드업, 기획, 고객 콜, 스프린트 플래닝)
- Fireflies.ai 사용 중이나 AI 크레딧 소진으로 자주 이탈
- 회의 요약을 Slack에 붙여넣지만 팀원 반응 없음

**핵심 불만**
- "요약 텍스트를 공유해도 아무도 안 읽는다"
- 액션아이템이 텍스트 중간에 묻혀 놓치는 일이 잦다
- 고객 콜 내용을 로드맵 근거로 엔지니어에게 설명하는 데 매번 시간 낭비

**IdeaTracer 사용 시나리오**
1. 스프린트 플래닝 후 → 결정사항 마인드맵을 팀 채널에 공유
2. 고객 콜 마인드맵을 로드맵 결정 근거로 직접 첨부 (Jira 카드에 링크)
3. 크로스 미팅 뷰: 3개월치 고객 콜에서 반복되는 요청 패턴 추적

**구매 패턴**: 팀 플랜 구매. 협업 공유 기능이 핵심 구매 조건. 제품 적합도 **높음**.

### 4.4 페르소나 3 — 비즈니스 컨설턴트 "이민주"

**프로필**: 41세, 독립 비즈니스 컨설턴트, 클라이언트 7~8곳 동시 관리

**현재 워크플로우**
- 클라이언트 인터뷰·워크숍 주 8~10건, 대면 포함
- 클로바노트 전사 → Xmind 마인드맵 수동 작성 → 보고서 첨부
- 이 2단계 툴 전환에 건당 **1~2시간 낭비**

**핵심 불만**
- "클로바노트 보고 Xmind 켜는 게 습관이 돼버렸어. 이게 합쳐지면 진짜 쓸 것 같아"
- 클라이언트에게 텍스트 보고서보다 시각 자료 제공을 선호하지만 자동화 수단이 없다
- 대면 녹음 파일 처리가 온라인 대비 현저히 불편

**IdeaTracer 사용 시나리오**
1. 대면·온라인 인터뷰 파일 업로드 → 즉시 마인드맵 생성
2. 마인드맵을 클라이언트 보고서 첨부 자료로 바로 활용
3. 툴 전환 없이 전사 → 시각화 → 공유가 단일 플로우로 완성

**구매 패턴**: 개인 고단가 플랜. 한국어 음성 인식 품질이 최종 결정 변수. 제품 적합도 **중상**.

## 5. 핵심 기능 정의

> 기능은 세 레이어로 구분한다: **Must Have (MVP)** / **Should Have (v1)** / **Nice to Have (v2+)**

### 5.1 Layer 1 — Must Have (MVP, 0→1 검증용)

#### F-01. 실시간 회의 봇
- Zoom, Google Meet, Microsoft Teams 자동 참여
- 다국어 음성 인식 (한국어 우선 최적화 + 영어)
- 화자 분리(Speaker Diarization) 자동 처리
- 회의 종료 즉시 처리 시작 (5분 이내 마인드맵 생성)

#### F-02. AI 마인드맵 자동 생성
- 전사본에서 주제·서브주제·세부 내용 계층 자동 추출
- 노드별 중요도 자동 가중치 (발언 빈도, 강조 표현, 반복 키워드 기반)
- 의사결정 / 액션아이템 / 질문 / 아이디어 유형 자동 태깅
- 인터랙티브 노드 클릭 → 원본 음성 타임스탬프로 즉시 이동

#### F-03. 오디오 파일 업로드 지원
- mp3, mp4, m4a, wav, webm 포맷 지원
- 대면 녹음 파일 처리 (클로바노트 대안)
- 배치 업로드: 최대 5개 파일 동시 처리

#### F-04. 마인드맵 기본 편집 & 공유
- 노드 추가/삭제/이동/이름 변경
- 공유 링크 생성 (뷰어 전용, 편집 가능 권한 분리)
- PNG/PDF/Markdown 내보내기

### 5.2 Layer 2 — Should Have (v1 정식 출시)

#### F-05. 크로스 미팅 인사이트
- 여러 회의에 걸쳐 반복되는 키워드·주제 자동 감지
- "이 키워드가 처음 등장한 회의" 타임라인 뷰
- 주간/월간 회의 인사이트 요약 대시보드

#### F-06. 화자별 기여도 시각화
- 화자별 발화 시간 비율 차트
- 화자별로 제기한 주제·이슈·질문 분류 뷰
- 회의 참여도 히트맵 (시간대별 참여 강도)

#### F-07. 액션아이템 추적
- AI가 추출한 액션아이템 자동 목록화
- 담당자·기한 태깅
- Notion, Jira, Slack 연동 내보내기

#### F-08. 마인드맵 커스터마이즈
- 레이아웃 변경 (방사형 / 트리형 / 타임라인형)
- 색상·폰트 테마 설정
- 팀 브랜드 컬러 적용

### 5.3 Layer 3 — Nice to Have (v2, 2026년 하반기+)

#### F-09. 실시간 마인드맵 (회의 중 라이브 생성)
#### F-10. 감정·온도 타임라인
#### F-11. 멀티 인터뷰 패턴 분석 (UX 리서처 특화)
#### F-12. API & 웹훅 제공

> **Who** 전사 텍스트를 읽는 데 지치고, 마인드맵을 수동으로 만드는 데 시간을 낭비하는 사람
> **IdeaTracer is** 회의가 끝나는 즉시 탐색 가능한 마인드맵을 자동 생성하는 AI 플랫폼
> **Unlike** Otter/Fireflies처럼 텍스트 덩어리만 주는 도구, 또는 Mapify/Xmind처럼 수동 업로드가 필요한 도구
> **IdeaTracer** 는 봇, 전사, 시각화, 타임스탬프 연결을 하나의 플로우로 완성한다

## 7. 기술 아키텍처 방향

### 7.1 핵심 기술 스택 방향

```
[회의 플랫폼] ──봇 API──▶ [IdeaTracer 봇 레이어]
                              │
                         [STT 엔진]
                         ├── 한국어: Clova/HyperCLOVA 또는 커스텀 파인튜닝
                         └── 영어: Whisper large-v3 또는 Deepgram
                              │
                         [LLM 분석 레이어]
                         ├── 주제 추출 & 계층화
                         ├── 의사결정/액션아이템 분류
                         └── 중요도 가중치 산정
                              │
                         [마인드맵 렌더러]
                         ├── 인터랙티브 노드 그래프
                         ├── 타임스탬프 인덱스 연결
                         └── 실시간 업데이트 (v2)
```

### 7.2 핵심 기술 의사결정 사항

| 결정 사항 | 옵션 A | 옵션 B | 권장 방향 |
|----------|--------|--------|----------|
| STT 엔진 | OpenAI Whisper (자체 호스팅) | 클로바 API + Deepgram | 초기: Whisper + Clova API 병행 |
| LLM | GPT-4o | Claude 3.5 Sonnet | 양쪽 테스트 후 비용/품질 최적화 |
