## 1. 화면 목적 재정의

### 기존 (v1) 목적
> "회의 내용을 마인드맵으로 시각화한다"

### 재정의된 (v2) 목적
> "회의 구조를 파악하고, 특정 논의를 빠르게 찾아, 바로 업무로 연결한다"

이 차이가 이후 모든 UX 결정의 기준이 된다.

## 2. 핵심 UX 원칙

### 원칙 1 — 구조는 마인드맵, 탐색은 타임라인, 실행은 패널

| 영역 | 역할 | 사용자 행동 |
|------|------|-------------|
| **마인드맵** (중앙) | 회의 전체 구조 파악 | "어떤 주제들이 있었지?" |
| **타임라인** (하단) | 특정 논의 위치 탐색 | "그 얘기 어디서 나왔지?" |
| **사이드패널** (우측) | 세부 확인 + 업무화 | "이걸 누가 하기로 했지?" |

### 원칙 2 — 노드는 발언이 아니라 판단 단위

기존 방식인 **발언 1개 = 노드 1개**는 주제가 많아지면 필연적으로 무너진다.

**v2 노드 계층:**
```
회의/프로젝트 (루트)
└── 주제 (1차) — 최대 6~8개
    └── 핵심 논점 / 결정 / 미해결 쟁점 (2차) — 주제당 3~5개
        └── 액션아이템 (3차) — 담당자 + 기한 포함
```

- 일반 발언은 노드로 올리지 않는다. 2차 이상에 해당할 때만 노드화
- 같은 주제가 재등장하면 새 가지를 만들지 않고 기존 주제 노드에 병합
- 발언 원문은 노드 클릭 시 사이드패널에서 확인 (+ 오디오 점프)

### 원칙 3 — 재생과 마인드맵은 항상 동기화

오디오 재생 위치 → 현재 주제 노드 자동 하이라이트
노드 클릭 → 오디오 해당 시점으로 이동
타임라인 마커 클릭 → 오디오 이동 + 마인드맵 해당 주제 강조

## 4. 컴포넌트 스펙

### 4.1 마인드맵 노드 계층

#### 루트 노드 (회의 제목)
- 중앙 고정, 드래그 불가
- 회의 제목 + 날짜 + 전체 길이 표시

#### 주제 노드 (1차)
- 현재 TopicNode 스타일 유지
- 추가: 재생 구간 진입 시 테두리 강조 (인디고 글로우)
- 추가: 접기/펼치기 버튼 (주제 많을 때 관리)
- 최대 8개 → 초과 시 "기타 논의" 노드로 묶기

#### 논점/결정 노드 (2차)
- 새 노드 타입 `PointNode` 추가
- 유형 배지: `💡 논점` / `⚖️ 결정` / `❓ 미해결` / `⚡ 쟁점`
- 중요도에 따라 노드 크기 변화

#### 액션아이템 노드 (3차, 기존 ActionNode 개선)
- 담당자 + 기한 + 상태 (미완료 / 완료) 표시
- 상태 토글 클릭 가능

### 4.2 하단 오디오 플레이어 (탐색 컨트롤 바)

기존 단순 플레이어를 **탐색 컨트롤 바**로 확장.

```
┌──────────────────────────────────────────────────────┐
│ [▶]  00:00 ──────────────────────────── 08:45  [⚙]  │
│            ↑주제A  ↑주제B  ↑결정  ↑액션             │
│            03:12   05:40   07:10  07:55              │
└──────────────────────────────────────────────────────┘
```

#### 시크바 위 마커 2종

| 마커 | 시각 | 클릭 동작 |
|------|------|----------|
| **주제 전환** | 큰 점 + 주제명 라벨 | 해당 시점 재생 + 마인드맵 주제 강조 |
| **결정/액션** | 작은 다이아몬드 | 해당 시점 재생 + 사이드패널에 내용 표시 |

#### 현재 위치 표시
- 재생 헤드 위치에 맞는 주제 이름을 타임바 위에 실시간 표시
- 예: `📍 주제 B — 데이터 트래킹 논의 중`

### 4.3 사이드패널 개선

#### 현재
단일 발언 상세 보기

#### v2 탭 구조

| 탭 | 내용 |
|----|------|
| **발언** | 현재 선택 발언 원문 + 오디오 점프 (기존 유지) |
| **결정** | 이 회의에서 확정된 사항 목록 |
| **액션** | 담당자 · 기한 · 상태가 있는 할 일 목록 |
| **쟁점** | 합의되지 않은 이슈 목록 |

#### 업무화 버튼 (각 탭 공통)
- `Notion으로 내보내기` (기존)
- `Markdown 복사` (기존)
- `클립보드에 복사` (신규)

## 5. 인터랙션 흐름

### 5.1 재생 → 마인드맵 동기화
```
오디오 재생 중
  → AudioPlayer onTimeUpdate(ms)
  → 현재 ms가 속한 Topic 계산
  → 해당 TopicNode 강조 (링 애니메이션)
  → 타임바 위에 현재 주제명 표시
  → 주제 전환 시점: 새 마커에 도달하면 pulse 효과
```

### 5.2 노드 클릭 → 오디오 이동
```
노드 클릭
  → 해당 노드의 대표 발언 startMs 계산
  → AudioPlayer seek(ms)
  → SidePanel에 관련 발언 목록 표시
```

### 5.3 타임라인 마커 클릭 → 전체 동기화
```
주제 전환 마커 클릭
  → seek(markerMs)
  → TopicNode 강조
  → SidePanel → 해당 주제 요약으로 전환
```

## 7. 구현 우선순위

| 우선순위 | 항목 | 난이도 | 이유 |
|----------|------|--------|------|
| P0 | 하단 탐색 컨트롤 바 (시크바 + 주제 마커) | 중 | 탐색 UX 핵심, 지금 없으면 서비스 완성도 ↓ |
